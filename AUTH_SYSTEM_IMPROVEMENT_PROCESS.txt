PHASE 7: SECURITY HARDENING
================================================================================

7A: GOOGLE LOGIN SECURITY FIX

PROBLEM:
The Google login flow trusted client-supplied uid and email.
A malicious user could POST fake uid/email to impersonate any user.

BEFORE (INSECURE):
  Frontend: signInWithPopup -> sends { uid, email } to server
  Server: trusts uid/email, looks up user, logs them in

AFTER (SECURE):
  Frontend: signInWithPopup -> gets ID token -> sends { id_token }
  Server: verifies ID token via Firebase Admin SDK -> extracts
          verified uid, email, photoURL from the token

FILES MODIFIED:
- app/Http//Auth/GoogleAuthController.php
  Both handleCallback() and linkAccount() now:
  1. Accept 'id_token' instead of 'uid'/'email'
  2. Verify the token via $firebaseAuth->verifyIdToken()
  3. Extract uid/email/picture from verified token claims
  4. Return 401 if token verification fails
  Uses Kreait\Firebase\Auth (kreait/laravel-firebase package)

- resources/views/auth/login.blade.php
  Frontend now calls result.user.getIdToken() and sends id_token

- resources/views/student/dashboard.blade.php
  Google linking now sends id_token instead of uid/photoURL

- resources/views/professor/dashboard.blade.php
  Google linking now sends id_token instead of uid/photoURL

7B: DEAD CODE CLEANUP

DELETED:
- app/Http/Controllers/Auth/RegisteredUserController.php
  Had broken syntax (Rules->Password::defaults())
  create() and store() methods were unrouted dead code
  Only checkStudent() was used (via /api/check-student/{code})

MOVED:
- checkStudent() method moved to StudentRegistrationController.php
- Route updated in web.php to point to StudentRegistrationController

CLEANED:
- app/Http/Controllers/Auth/QrLoginController.php
  Removed ~96 lines of commented-out duplicate class (lines 3-96)
  Removed dead showQrForm() method (was overridden by auth.php's
  AuthenticatedSessionController@create)
  Kept only: handleScan(), finalizeLogin(), refreshQr()

7C: DUPLICATE ROUTE FIX

PROBLEM:
GET /login was defined twice:
1. web.php line 75: Route::get('/login', [QrLoginController::class, 'showQrForm'])
2. auth.php line 15: Route::get('login', [AuthenticatedSessionController::class, 'create'])

Since auth.php is required at the end of web.php, the auth.php version wins.
Both generate QR codes and show the login view, so they're functionally identical.

FIX:

PHASE 3: OTP VERIFICATION SYSTEM
================================================================================

PROBLEM:
No second-factor verification after registration.
Laravel's built-in email verification (signed URL) was not
being used effectively. Students were auto-logged in without
any verification step.

SOLUTION:
5-digit OTP code system with email and Telegram delivery.

DATABASE:
Created migration: 2026_06_26_154502_add_otp_and_verification_to_users_table.php
Adds columns:
  - otp_code (nullable string)
  - otp_expires_at (nullable timestamp)
  - otp_attempts (integer, default 0)
  - otp_last_sent_at (nullable timestamp)
  - is_verified (boolean, default false)
  - verification_method (nullable string: 'email' or 'telegram')

FILES CREATED:
- app/Services/OtpService.php
  Core OTP logic:
    OTP_LENGTH = 5
    OTP_EXPIRY_MINUTES = 10
    OTP_MAX_ATTEMPTS = 5
    OTP_RESEND_COOLDOWN_SECONDS = 60

    Methods:
    - generateOtp(): random 5-digit zero-padded string
    - sendOtp(user, method): generates OTP, stores in DB, sends via
      email or Telegram, enforces cooldown via Cache
    - verifyOtp(user, otp): checks attempts, expiry, hash_equals,
      increments attempts on failure, clears OTP on success
    - getRemainingCooldown(user): seconds until resend allowed
    - getRemainingAttempts(user): attempts left

    Email delivery: Laravel Mail::raw()
    Telegram delivery: HTTP POST to api.telegram.org/bot{token}/sendMessage

- app/Http/Controllers/Auth/OtpController.php
  Controller with methods:
    - show(): displays OTP verification page (redirects if already verified)
    - verify(request): validates 5-digit OTP, calls OtpService
    - resend(): sends new OTP, supports AJAX (JSON) and web responses
    - showLookup(): displays check-verification page
    - lookup(): finds user by email/phone/student_id_code,
      redirects to OTP if not verified, shows info if already verified
    - getMaskedContact(user): returns masked email or phone
    - maskPhone(phone): shows first 3 and last 2 digits
    - maskEmail(email): shows first and last char of local part

- resources/views/auth/verify-otp.blade.php
  Styled OTP verification page with:
    - 5 individual digit inputs with auto-advance
    - Paste support (auto-distributes digits)
    - Backspace navigation
    - Hidden full OTP input for form submission
    - Green filled state on completed digits
    - Resend button with countdown timer
    - Back to login link
    - Toast notifications for success/error

FILES MODIFIED:
- routes/auth.php
  Added OTP routes in auth middleware group:
    GET  /verify-otp        -> OtpController@show        (otp.show)
    POST /verify-otp        -> OtpController@verify      (otp.verify)
    POST /verify-otp/resend -> OtpController@resend      (otp.resend)
  Added check-verification routes in guest middleware group:
    GET  /check-verification -> OtpController@showLookup  (check-verification)
    POST /check-verification -> OtpController@lookup      (check-verification.lookup)

- app/Models/User.php
  Added OTP fields to $fillable:
    'otp_code', 'otp_expires_at', 'otp_attempts',
    'otp_last_sent_at', 'is_verified', 'verification_method'
  Added OTP fields to casts:
    'otp_expires_at' => 'datetime'
    'otp_last_sent_at' => 'datetime'
    'is_verified' => 'boolean'

- app/Http/Controllers/StudentRegistrationController.php
  After successful registration:
    1. Sets is_verified = false on the user
    2. Calls OtpService::sendOtp(user, 'email')
    3. Redirects to OTP verification page instead of dashboard
    4. Shows success message: "Check your email for verification code"

- app/Http/Controllers/Auth/AuthenticatedSessionController.php
  After successful login:
    Checks if user->is_verified is false
    If not verified, redirects to otp.show instead of dashboard

- app/Http/Controllers/Auth/QrLoginController.php
  In finalizeLogin():
    After Auth::loginUsingId(), checks is_verified
    If not verified, redirects to otp.show

- app/Http/Controllers/Auth/GoogleAuthController.php
  In handleCallback():
    After Auth::login(), checks is_verified
    If not verified, returns JSON with status='unverified' and redirect URL

- resources/views/auth/login.blade.php
  Google login handler now checks for data.status === 'unverified'
  and redirects to data.redirect URL

RESULT:
After registration, users receive a 5-digit OTP via email.
They must enter it on the verification page before accessing the system.
5 max attempts, 10-minute expiry, 60-second resend cooldown.
Unverified users are redirected to OTP page on every login attempt.
